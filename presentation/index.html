Double UTF-8

Program A produces some Unicode text.
It sends it to Program B in UTF-8.
>>> text = "Olá, mundo"
>>> text.encode('utf-8')
b'Ol\xc3\xa1, mundo'
Program B thinks bytes are the same as characters.
(It's probably not written in Python 3.)
>>> bytestring = b'Ol\xc3\xa1, mundo'
>>> bytestring.decode('windows-1252')

CESU-8

What happens when you encode text in UTF-16, pretend the result is still Unicode, and encode that in UTF-8.

Popularized by Java and MySQL.

Python 3 is not such a fan.

>>> bytestring = b'This is awesome \xed\xa0\xbd\xed\xb8\x84'
>>> bytestring.decode('utf-8')
UnicodeDecodeError: 'utf-8' codec can't decode byte 0xed in position 16: invalid continuation byte

>>> from ftfy import bad_codecs
>>> bytestring.decode('utf-8-variants')
'This is awesome 😄'

# That's a grinning face. Thanks, JavaScript.
